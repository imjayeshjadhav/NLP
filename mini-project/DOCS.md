# Project Documentation — Transformer-Based Text Summarization

## Overview

This project implements an end-to-end **abstractive text summarization** system using pre-trained Hugging Face transformer models (BART and T5). It includes a text preprocessing pipeline, model inference wrappers, ROUGE-based evaluation, a fine-tuning driver, and a Streamlit web UI for interactive use.

## How It Works

### 1. Text Preprocessing — `src/preprocessing.py`

Raw input text is cleaned before being fed to any model.

**Steps applied by `clean_text()`:**
1. Unicode normalization (NFKC) — collapses ligatures and special widths
2. Strip control characters
3. Remove URLs and bracketed citations (e.g., `[1]`)
4. Keep only alphanumerics and standard punctuation
5. Collapse repeated whitespace

**Model-specific preprocessing via `preprocess_for_model()`:**
- T5 requires a task prefix: input becomes `"summarize: <cleaned text>"`
- BART receives the cleaned text directly

---

### 2. Summary Generation — `src/summarizer.py`

Three models are supported, each with a dedicated public function:

| Function | Model | HF Identifier |
|---|---|---|
| `generate_summary_bart()` | BART | `facebook/bart-large-cnn` |
| `generate_summary_t5()` | T5 | `google/flan-t5-small` |
| `generate_summary_fine_tuned()` | Fine-tuned T5 | `models/t5-finetuned/` |

**Key design decisions:**
- Models are loaded lazily on first call and cached in a module-level dict — no re-loading between Streamlit reruns.
- If no fine-tuned checkpoint exists, `generate_summary_fine_tuned()` falls back to base T5 automatically.
- All generation uses beam search with `length_penalty=2.0`, `no_repeat_ngram_size=3`, and `early_stopping=True`.
- Every call returns a `SummaryResult` dataclass containing: `summary`, `input_length`, `summary_length`, and `inference_time_s`.

**Generation parameters (configurable from UI):**

| Parameter | Default | Description |
|---|---|---|
| `max_length` | 150 | Hard cap on output tokens |
| `min_length` | 40 | Minimum output tokens |
| `num_beams` | 4 | Beam search width |

---

### 3. Evaluation — `src/evaluation.py`

ROUGE-1, ROUGE-2, and ROUGE-L F1 scores are computed against an optional reference summary.

- **Primary backend:** `evaluate.load("rouge")` (Hugging Face)
- **Fallback backend:** `rouge_score.RougeScorer` (used if `evaluate` is unavailable)

**Key functions:**

| Function | Description |
|---|---|
| `compute_rouge(prediction, reference)` | Single-pair ROUGE scores |
| `compute_rouge_batch(predictions, references)` | Corpus-level mean ROUGE |
| `build_comparison(summaries, reference)` | Builds a comparison table with length, clarity, redundancy, and ROUGE per model |
| `plot_rouge_scores(rows)` | Returns a matplotlib grouped bar chart of ROUGE scores |

**Clarity and redundancy** are heuristic scores computed from token statistics — they serve as lightweight quality proxies when no reference is available.

---

### 4. Fine-Tuning — `src/fine_tune.py`

Fine-tunes `google/flan-t5-small` on a small slice of the **CNN/DailyMail** dataset using the Hugging Face `Seq2SeqTrainer`.

**Default configuration:**

| Parameter | Default |
|---|---|
| Training samples | 200 |
| Eval samples | 50 |
| Epochs | 1 |
| Batch size | 2 |
| Learning rate | 5e-5 |
| Max input tokens | 512 |
| Max target tokens | 128 |

**Process:**
1. Load dataset slices from `cnn_dailymail` (3.0.0)
2. Tokenize with `"summarize: "` prefix
3. Compute baseline ROUGE on eval set (before training)
4. Run `Seq2SeqTrainer.train()`
5. Save model + tokenizer to `models/t5-finetuned/`
6. Compute post-training ROUGE and log the before/after delta

**Run command:**
```bash
python -m src.fine_tune --train_size 200 --eval_size 50 --epochs 1
```

---

### 5. Streamlit UI — `app.py`

The web interface ties all components together.

**Layout:**
- Sidebar: model selector (`BART`, `T5`, `Fine-Tuned T5`, `Compare All`) and generation parameter sliders
- Main area: article input + optional reference summary input
- Expander: before/after text cleaning preview
- Output: generated summary with word count, compression ratio, inference time, and ROUGE scores (if reference provided)
- Comparison mode: side-by-side table and ROUGE bar chart across all three models

**Run command:**
```bash
streamlit run app.py
```

## Quickstart

```bash
# 1. Set up environment
python -m venv .venv
.venv\Scripts\activate          # Windows
# source .venv/bin/activate     # macOS/Linux

# 2. Install dependencies
pip install -r requirements.txt

# 3. Launch the UI
streamlit run app.py

# 4. (Optional) Fine-tune T5
python -m src.fine_tune --train_size 200 --eval_size 50 --epochs 1
```

Open http://localhost:8501 in your browser. The first run downloads model weights (~300 MB for T5, ~1.6 GB for BART).

## Expected ROUGE Results

Measured on 50 CNN/DailyMail validation samples with T5-small:

| Phase | ROUGE-1 | ROUGE-2 | ROUGE-L |
|---|---|---|---|
| Base T5 (no fine-tuning) | ~0.22 | ~0.08 | ~0.18 |
| After 1 epoch (200 samples) | ~0.27 | ~0.11 | ~0.22 |

BART scores higher out-of-the-box as it is already pre-trained on CNN/DailyMail. The fine-tuning experiment demonstrates a measurable ROUGE lift even with a minimal training budget.
